# QQQ · 趋势投资者的"放假分界线"

> 一个简单的想法：长持 QQQ 已经够好。但如果能在最惨的几次熊市里"放假"——空仓躲一阵——是不是收益会更好？
> 这个项目用 26 年的 QQQ 历史数据回测了这个问题，并给你一个**可拖动滑块**的网页，自己找答案。

## 核心问题

如果只在 QQQ 收盘价 ≥ `MA200 × k` 时持仓，跌破就空仓——

- `k` 设多大，收益最高？
- 历史上需要"放假"多少天？
- 这种策略到底是真有用，还是只在事后看着好看？

## 策略逻辑

```
若  前一日收盘价 ≥ MA200 × k  →  今日全仓 QQQ
若  前一日收盘价 <  MA200 × k  →  今日空仓（放假）
```

- 200 日均线 (MA200) 是中长期趋势的标尺
- 乘以系数 `k` 把"分界线"上移或下移：
  - `k = 1.00` → 经典"跌破均线就跑"
  - `k = 0.83` → 跌破均线后再跌 17% 才跑（只在大熊市触发）
  - `k = 1.05` → 强势线之下就跑（非常敏感）
- 信号用 **前一日** 收盘价，避免未来函数

## 历史结论（截至 2026-04）

| k 值 | 放假比例 | 总收益 | 年化 | vs 长持 |
|------|---------|--------|------|--------|
| 长持 QQQ（基准） | 0% | +760% | **8.54%** | — |
| 0.70 | 2.6% | +328% | 5.69% | -2.85% |
| 0.83 | 7.8% | **+1260%** | **10.45%** | **+1.91%** ✅ |
| 0.90 | ~13% | ~+1050% | ~9.7% | +1.2% |
| 0.97 | 20.6% | +771% | 8.59% | +0.05% |
| 1.00 | 26.5% | +628% | 7.85% | -0.69% |
| 1.05 | 42.8% | +328% | 5.73% | -2.81% |

**两个反直觉的发现：**

1. **简单"跌破 MA200 就跑"反而跑输长持** —— QQQ 一年里大约 ¼ 时间在均线下方反复穿越，频繁假信号让你不停高买低卖，错过反弹。
2. **"放假分界线"有一个清晰的甜点（0.82–0.86）** —— 比这窄触发不到熊市，比这宽被噪音吃掉收益。这条线只在 2000-2002、2008-2009、2022 这几次真熊市才触发，全年只让你"放假" 8% 的时间，但避开了最深的尾部下跌。

## ⚠️ 该被怀疑的地方

- 触发样本极少：26 年只有 ~514 个"放假"日，集中在 3 次熊市。所谓"甜点"很可能是**对历史的过拟合**，未必能复制到未来。
- 不含成本：实盘有交易费、税、滑点。频繁切仓的策略实际表现会更差。
- 阻力路径：在熊市中坚持空仓不抄底，比回测里点击一下难得多。

把它当成**一个直觉校准工具**，不是交易系统。

## 用法

### 本地运行

```bash
git clone https://github.com/<you>/qqq-vacation.git
cd qqq-vacation
pip install yfinance pandas
python build.py
```

生成 `index.html`，用浏览器打开即可。所有数据已嵌入，无需联网。

### 自动每日更新（GitHub Actions）

仓库已配置 `.github/workflows/update.yml`，每天 UTC 01:00（美东收盘后）自动：

1. 拉取最新 QQQ 数据
2. 重新生成 `index.html`
3. 推送到 `main` 分支

只需在 GitHub 仓库设置里开启 **Pages**（Source = `main` / root），就有一个永远最新的在线版本。

### 调整阈值范围

`index.html` 里滑块默认 `0.70 ~ 1.05`，步长 `0.005`。要改的话编辑 `build.py` 模板里的 `<input type="range">` 即可。
